# CONTEXTUAL_UI_STANDARD.md

Статус: активне джерело правди.

Контекстний UI дозволений, коли він справді залежить від поточного вибору або режиму.

## Дозволені патерни

- inline mini-toolbar;
- context menu;
- inspector;
- sticky strip;
- popover з параметрами.

## Правила

- Не дублювати весь toolbar.
- Не приховувати критичні для молодших учнів параметри в нестабільний floating UI.
- Не зсувати workspace без потреби.
- Не створювати кілька різних способів виконати одну дію, якщо це не прискорює реальний сценарій.
- Якщо дія стандартна (`new/open/save/undo/redo`), вона має йти через command adapter, а не через контекстний виняток.

## Перевірка

Контекстний UI переважно перевіряється ручним QA. Статичний аудит ловить лише базові контракти shell, команд, modal/dropdown і стилів.
