# UI_MIGRATION_TO_STANDARD.md — ПЛЮС Слайди

## Поточний Стан

Базовий shell, command adapter, file picker і runtime-аудіт підключені. `slides/js/runtime.js` лишається стабільним module entrypoint, а `slides/js/app.js` тепер є координатором верхнього рівня, не єдиним місцем усієї логіки редактора.

Поточний прохід зменшив технічний борг без надмірного збільшення кількості файлів: з `app.js` винесено тільки ті частини, які мають самостійну відповідальність і потрібні для майбутнього PowerPoint-подібного функціоналу.

## Поточна Структура

- `slides/js/runtime.js` — module entrypoint, який стійко запускає `SlidesApp.boot`.
- `slides/js/app.js` — boot, UI coordinator, command dispatch і зв'язування модулів.
- `slides/js/project.js` — нормалізація презентації/елементів, import/export JSON payload, filename slug.
- `slides/js/slide-list.js` — thumbnails, drag reorder і кнопки керування слайдами.
- `slides/js/stage-renderer.js` — DOM-рендеринг сцени, елементів, resize/rotate handles і selected-state.
- `slides/js/text-list.js` — створення, рендеринг і читання маркованих/нумерованих текстових блоків.
- `slides/js/stage-interactions.js` — pointer state, drag/resize/rotate, рамка вибору (marquee), smart guides + прив'язка (екранний поріг, AABB з урахуванням rotation), rotation-aware resize.
- `slides/js/modal-ui.js` — show/close/info/confirm modal behavior.
- `slides/js/state.js` — локальний state і serialization.
- `slides/js/history.js` — undo/redo stack.
- `slides/js/storage.js` — autosave draft у IndexedDB з graceful-фолбеком на localStorage і міграцією старої localStorage-чернетки.
- `slides/js/templates.js` — створення слайдів, елементів і макетів.
- `slides/js/export.js` — PDF export, print і snapshot rendering.
- `slides/js/utils.js` — DOM, file і utility helpers.

## Реалізовано (хвилі 0–4.2)

- **Хвиля 0 — надійність ядра:** коректний undo/redo (commit-after, коаліс­ція drag/typing), розділені статуси `dirty`/draft-saved/file-saved, захищена versioned-нормалізація імпорту (ліміти, allowlist кольорів, нейтралізація зовнішніх URL, унікальні ID), автозбереження в IndexedDB з фолбеком на localStorage і реконсиляцією за часом.
- **Хвиля 1 (частково) — мультивибір:** `selectedElementIds`, shift-клік, Ctrl+A, marquee, груповий drag/нудж/delete/copy/paste, форматування за всіма вибраними. (№6 align/distribute/group — ВІДКЛАДЕНО.)
- **Хвиля 2 — зручне полотно:** zoom (фіксована сцена 960×540 + transform-scale, fit-to-window, Ctrl+±/0, Ctrl+колесо); smart guides + прив'язка до країв/центра/об'єктів + сітка; rotation handle, Shift-пропорції resize, rotation-aware resize, Alt+drag-дублювання, контекстне меню (миша + Shift+F10); доступність (alt text зображень, видимий focus, Tab-цикл об'єктів, ↑/↓ між слайдами, aria-labels).
- **Хвиля 3.1 — базове форматування текстового блока:** локальні сімейства шрифтів, довільний розмір 4–400, довільний міжрядковий інтервал 0.8–3 (зберігається при відкритті/відновленні), нормальна/жирна вага 400/700; однаковий рендер на полотні, у мініатюрах і PDF/print; allowlist-нормалізація нових полів та версія текстової моделі у файлі.
- **Хвиля 3.2 — списки текстового блока:** марковані й нумеровані списки з перемиканням у toolbar, семантичним `ul`/`ol`-рендером на полотні, у мініатюрах і PDF/print, Undo/Redo та allowlist-нормалізацією `listType`.
- **Хвиля 3.3 — заміна зображення:** заміна вибраного зображення з файла або URL зі збереженням його `id`, позиції, розміру, повороту, шару й вибору; alt можна зберегти або змінити; операція підтримує Undo/Redo. HTTPS-посилання вбудовуються в `data:` одразу при вставці/заміні (fetch+encode), тож у моделі зберігається лише `data:` — зображення переживає перезбереження й працює офлайн, а сирі зовнішні URL не потрапляють у файл; невалідне джерело лишає модалку відкритою з інлайн-поясненням.
- **Хвиля 3.4 — оформлення зображення:** режим вписування в рамку (`cover`=заповнити з обрізанням / `contain`=вписати повністю) і прозорість 0–100% («прозорість», 0%=видиме); неруйнівне **кадрування** (crop) — частки рамки `l/t/r/b`, режим із crop-ручками й затемненим «привидом» обрізаного, вихід Esc/Enter, Undo/Redo; окрема група тулбара з'являється лише коли вибрано зображення; однаковий рендер (через вікно `overflow:hidden`, без `clip-path`) на полотні, у мініатюрах і PDF/print; allowlist-нормалізація `objectFit`, clamp `opacity` та `crop`.
- **Хвиля 3.5 — базові лінії та стрілки:** вставка з меню й тулбара, allowlist-нормалізація, однаковий SVG-рендер на полотні, у мініатюрах і PDF/print; вибір, drag, зміна довжини зліва/справа, rotation, шари, копіювання та Undo/Redo працюють через спільну модель `shape`.
- **Хвиля 3.6 — текст у фігурі:** прямокутник, коло й трикутник зберігають напис у власному `shape.content`; один клік/drag працює з фігурою, подвійний клік входить у редагування; доступні звичайне текстове форматування, списки, Undo/Redo та однаковий рендер на полотні, у мініатюрах і PDF/print. Службові placeholder-и видно в редакторі та мініатюрах, але вони не потрапляють у показ, PDF чи друк. Лінії та стрілки текст не підтримують; файловий контракт позначено `textModelVersion: 3`, щоб старі версії не втрачали нові написи мовчки.

- **Хвиля 4.1 — теми оформлення (дизайн-токени):** набір тем (`THEMES`: фон слайдів + узгоджена акцентна палітра, лише локальні кольори для офлайну); picker у меню «Слайд → Тема оформлення…»; застосування оновлює фон УСІХ слайдів і колірну палітру popover, не чіпаючи вміст елементів; `theme` зберігається у файлі/чернетці, нормалізується через allowlist (невідома → `classic`), бере участь в Undo/Redo (serialize/applyPresentationData + renderColorPalette у renderAll).

- **№6 — вирівнювання, розподіл, групування (відкладений пункт Хвилі 1, ЗАКРИТО):** для мультивибору (≥2) — вирівнювання ліворуч/центр-гориз/праворуч/угору/середина-верт/донизу за спільним AABB (з урахуванням rotation); рівномірний розподіл центрів по горизонталі/вертикалі (≥3); **групування/розгрупування** (`element.groupId`, Ctrl+G/Ctrl+Shift+G) із «липким» вибором (клік по члену вибирає всю групу), ремап groupId при дублюванні/вставці (копія — окрема група); окрема група тулбара лише для мультивибору, розподіл активний від 3, «розгрупувати» — лише за наявності згрупованого; усе через Undo/Redo.
- **Хвиля 4.2 — макети слайдів і типізовані placeholder-и:** `LAYOUTS` (blank/title/title-body/two-content/title-image) із типізованими слотами (`title/subtitle/body/image`); picker «Слайд → Макет слайда…» застосовує макет до поточного слайда НЕРУЙНІВНО (реальний вміст лишається, порожні placeholder-и оновлюються, дублікати за типом не плодяться); `slide.layout` та `element.placeholderType` нормалізуються через allowlist і зберігаються; image-placeholder — порожній image-елемент (пунктирна рамка), заповнюється подвійним кліком (через replace-флоу, що знімає `isPlaceholder`); placeholder-и видно в редакторі та мініатюрах, але не в PDF/print/показі; Undo/Redo.

## Зафіксовані Перевірки

- `tests/run-tests.ps1` перевіряє статичний shell/module контракт (зокрема відповідність `$('#id')`-посилань HTML- id з allowlist динамічних).
- `tests/run-browser-smoke.ps1` (бюджет virtual-time 35000) запускає `tests/slides-behavior.html`.
- `tests/slides-behavior.html` (~190 перевірок) покриває: boot/рендер/стандартні команди; нормалізацію (ліміти, безпека, унікальність ID, alt, текстову модель); storage (IDB round-trip, quota, tombstone, reload-restore); undo/redo + коаліс­цію; мультивибір (Ctrl+A, multi-delete, shift-toggle, marquee, copy/paste, змішане форматування); zoom (масштаб, незмінність координат, drag/resize/marquee при zoom); smart guides + сітку; rotation/Shift-resize/rotation-aware resize/Alt-drag/контекст-меню; форматування і списки; заміну, вписування, прозорість і кадрування зображень; лінії/стрілки; текст у фігурах; snapshots без службових placeholder-ів; доступність (Tab-цикл, навігація слайдів, клавіатурне контекст-меню, aria-label).

## Обмеження

Не повторювати помилку надмірного дроблення. Новий модуль у `slides/js` має з'являтися лише тоді, коли він забирає самостійну відповідальність з `app.js`, додається в `sw.js` за потреби і покривається статичним аудитом або browser-smoke тестом.

Поточний висновок: Слайди достатньо стабілізовані для функціонального розвитку. Наступний редактор для техборгу перед нарощуванням можливостей — `paint/`.
