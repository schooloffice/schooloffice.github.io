# Єдиний UI-стандарт Офіс ПЛЮС

Статус: активне джерело правди.

Цей документ описує продуктові принципи. Технічні деталі інтеграції живуть у `UI_INTEGRATION_GUIDE.md`.

## Принципи

- Усі редактори мають виглядати як частини однієї екосистеми.
- Однакові дії мають бути в однакових місцях.
- Локальна специфіка редактора не повинна ламати shared shell.
- Часті дії розміщуються ближче, рідкісні дії йдуть у меню або popover.
- Інтерфейс має лишатися простим для школи і достатньо професійним для повсякденної роботи.

## Обов'язковий shell

Кожен редактор має:

1. header;
2. головне меню;
3. toolbar;
4. workspace;
5. statusbar.

Контекстні панелі, інспектори і popover-и дозволені лише тоді, коли вони не дублюють увесь toolbar.

## Стандартні команди

Обов'язкові команди:

- `new`
- `open`
- `save`
- `undo`
- `redo`

Вони мають бути узгоджені між:

- toolbar;
- головним меню;
- hotkeys;
- command adapter.

Кодовий контракт: `OfficeShell.registerCommands` / `OfficeShell.runCommand` як thin adapter-шар і `OfficeUI.registerCommands` / `OfficeUI.runCommand` як базовий shared shell API.

## Файли і збереження

Відкриття локального файлу має використовувати `OfficeShell.openFilePicker` або `OfficeUI.openFilePicker`.

Кожен редактор сам відповідає за:

- підтримувані формати;
- читання файлу;
- валідацію;
- імпорт у власну модель даних.

## Overlay

Dropdown, picker, popover і modal мають:

- підтримувати `Escape`;
- не залишати брудний локальний state;
- не ламати focus;
- не закривати модалку під час кліку всередині неї.

## Стилі

- Shared токени живуть у `UI_TOKENS.css`.
- Локальні `style.css` не перевизначають `--office-*`.
- Локальні `style.css` не перевизначають `.office-*` component selectors.
- Акцент сервісу має відповідати `SERVICE_THEME_MAP.json`.

## Перевірка

Перед завершенням змін запускати:

```powershell
powershell -ExecutionPolicy Bypass -File tests\run-tests.ps1
```

Браузерні тести використовувати точково, лише для поведінки, яку не можна надійно перевірити статично.
