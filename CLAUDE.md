# КРИТИЧНО ВАЖЛИВО — СТРОГИЙ РЕЖИМ ДЛЯ ШКІЛЬНИХ СТАТИЧНИХ САЙТІВ

Ти працюєш ВИКЛЮЧНО всередині папки `office/`. 
Це простий статичний сайт (HTML + CSS + JS) для учнів. 

ЗАБОРОНЕНО назавжди:
- Створювати git worktrees, копіювати репозиторій, створювати будь-які додаткові папки (.claude/worktrees тощо)
- Читати, сканувати або згадувати будь-які папки вище `office/` (включаючи корінь репозиторію, GitHub, compnauka.github.io тощо)
- Виконувати команди ls, find, grep, tree по всьому проєкту
- Читати файли "для контексту" або "щоб краще зрозуміти проєкт"
- Копіювати великий обсяг коду в контекст

ДОЗВОЛЕНО ТІЛЬКИ:
- Читати і редагувати файли, які я явно назву (наприклад: index.html, css/style.css, js/script.js)
- Працювати тільки з HTML, CSS, JavaScript

Правила роботи:
- Перед будь-якою дією запитуй: "Які конкретні файли мені потрібно прочитати для цього завдання?"
- Якщо завдання нечітке — став одне уточнююче питання.
- Пропонуй план з 2-3 кроків і чекай мого "OK".
- Відповідай коротко. Показуй тільки diff змін, а не весь файл.

Якщо потрібно щось поза office/ — відразу запитай мене.

# ВАЖЛИВО — межі роботи

Ти працюєш ВИКЛЮЧНО з папкою `office/` та її вмістом.

- НЕ читай, НЕ чіпай, НЕ переглядай інші папки репозиторію
- НЕ створюй worktree, НЕ копіюй вміст репозиторію
- НЕ виходь за межі папки `office/`
- НЕ створюй git worktrees НІКОЛИ. Працюй напряму з файлами в office/ без створення гілок або worktrees.
- Якщо завдання вимагає файлів поза `office/` — запитай мене спочатку
## Економія токенів

- Перед початком роботи запитай які КОНКРЕТНІ файли потрібні — не читай все підряд
- Читай лише файли, які я явно назвав або які безпосередньо потрібні для завдання
- НЕ роби `find`, `grep` по всій папці без мого дозволу
- НЕ читай файли "для контексту" якщо я не просив
- Якщо потрібен файл — назви його і запитай чи читати

## Стиль відповідей

- Відповідай коротко і по суті
- НЕ переказуй що ти зробив великими блоками тексту
- НЕ пояснюй очевидне
- Показуй тільки змінені частини коду, не весь файл повністю
- Якщо зміна велика — покажи diff, а не весь файл

## Перед кожним завданням

- Уточни завдання якщо воно нечітке — одне запитання, не п'ять
- Запропонуй план з 2-3 кроків і дочекайся мого підтвердження
- Не починай редагувати файли поки я не підтвердив план
Цей проєкт — офісний пакет для школи. Усі інші папки (apps, games, kids тощо) — непов'язані проєкти, не чіпай їх.
